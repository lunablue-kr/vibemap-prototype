# 서울 바이브맵 — 프로토타입 (와이어프레임 단계)

설계서 v0.4 기준 Phase 1 범위 프로토타입. **design-brief.md 확정 전이므로 본 UI 스타일링 없이
와이어프레임(회색 계열) + 서비스 로직까지만 구현** (CLAUDE.md 규칙).

## 실행

```
cd prototype && python3 -m http.server 8899
# 브라우저에서 http://localhost:8899
```

## 목(mock) 처리 — 실제 개발 시 교체 지점

- `js/mock-toss.js` → 토스 로그인·위치 SDK·기능성 메시지 (apps-in-toss-sdk 스킬 참조)
- `js/store.js` → Supabase (스키마는 설계서 §10 데이터 테이블 기준)
- 상단 노란 바 "위치 시뮬레이션" → 실기기에서는 startUpdateLocation

## 리뷰 결과 — 정식 개발 이관 체크리스트 (code/policy/design-reviewer 지적, 프로토타입에서 미해결)

**서버로 반드시 이전할 것 (Supabase 전환 시)** — 현재 전부 클라이언트 실행이라 조작 가능:
점수 계산, 하루 제한, 리액션 중복 방지, 신고 임계값, 모더레이션 판정, 주간 결산 배치(레벨업·이력 회전).

**정책 (변호사 확인 권장)**:
- 금칙어 사전에 혐오 표현·정치인 실명이 없음 — 사전 초안(설계서 §13)은 법률 자문 하에 구성
- 사람 조롱·상호명+부정 자동 감지 없음 (현재 신고 의존)
- 개인정보처리방침 미작성 (Phase 2 내돈내산 착수 전 필수)
- 보류(held) 대기열을 처리할 관리자 화면 없음 — held 태그는 현재 영구 비공개

**기능·UX**:
- 홈 지역구 무제한 변경 가능 (§13 인증 방식 미결 — 결정 후 쿨다운/GPS 인증 구현)
- 신고 사유 UI가 브라우저 prompt() — TDS 시트로 교체
- 태그 수백 개 규모 대비: 리액션 인덱스(Map), 마커 diff 렌더, 클러스터링 검토
- 구 상세 빈 화면 독려 문구 (copywriter 경유), 불씨 목록 가독성
- 출시 전 제거: 위치 시뮬레이션 바, 데이터 초기화 버튼, window.__vibemapMap
- 사전이 실분량으로 확정되면 /data/*.json 으로 외부화
