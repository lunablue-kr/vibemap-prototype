# 서울 바이브맵 — 프로토타입 v2 (와이어프레임 단계)

설계서 v0.5 기준 Phase 1 범위 프로토타입. design-brief.md 확정 전이므로 본 UI 스타일링 없이
와이어프레임(회색 계열) + 서비스 로직까지만 구현 (CLAUDE.md 규칙).

## v2 반영 (설계서 v0.5)

- IA 개편: 하단 탭 삭제, 지도 전체화면, 랭킹 롤링 칩(탭=랭킹 시트)·마이 아이콘·시트 구조
- 인터랙션: 태그 1탭=완전체 카드 팝업(○○구›/전문/리액션/⋯신고), 빈 공간 터치=인라인 작성창, 더블탭=줌
- 리액션 개편: 💪힘내 → ✨힙해요. 부문 "응원의" → "힙한 동네"
- 지도: 베이스 타일 제거(단색 배경+폴리곤), 자리싸움(크기 3단계·줌별 구당 상한·앵커·고정석·단일 갱신)
- 홈 인증: 최초 자유, 변경=현지 GPS 1회+쿨다운 28일 (home.js)
- 30자 제한, 점수 태그+3/리액션+1, 레벨 테이블 확정값, 승격 합산3·구내 상위10 (전부 config)
- 시드 25개 구 × 3태그

## 리뷰 결과 — 정식 개발 이관 체크리스트 (code/policy/design-reviewer 지적, 프로토타입에서 미해결)

**서버로 반드시 이전할 것 (Supabase 전환 시)** — 현재 전부 클라이언트 실행이라 조작 가능:
점수 계산, 하루 제한, 리액션 중복 방지, 신고 임계값, 모더레이션 판정, 주간 결산 배치(레벨업·이력 회전).

**정책 (변호사 확인 권장)**:
- 사전 v0.1 적용됨 (혐오·치안·집값·정치 커버). 정치인 실명은 운영 중 politics 배열에 수시 추가 방침 — 법률 자문 권장 유지
- 사람 조롱·상호명+부정 자동 감지 없음 (현재 신고 의존)
- 개인정보처리방침 미작성 (Phase 2 내돈내산 착수 전 필수)
- 보류(held) 대기열을 처리할 관리자 화면 없음 — held 태그는 현재 영구 비공개

**v2 리뷰 잔여 (디자인·정식 개발 단계 과제)**:
- 자리싸움 구 경계 클리핑 미구현 — 현재는 전역 픽셀 밀어내기 근사 (§9-3 "구 경계 안에서만"은 정식 구현 시)
- 구 레벨업·주간 결산 반영 로직 없음 — 서버 일요일 배치로 구현 예정 (레벨 테이블은 config 준비됨)
- ringCentroid는 MultiPolygon 시 첫 링만 사용 (서울 데이터는 전부 단일 Polygon이라 현재 무해)
- 보류(held) 태그가 작성자 본인에게도 안 보임 — "내 태그(검토 중)" 뷰 추가 여부 설계 판단 필요
- 연락처 차단 문구가 카테고리를 특정함("연락처나 외부 링크는…") — 우회 학습 방지 원칙과의 일관성 재검토
- 지도 태그 라벨 터치 높이 ~20px (44px 미달) — 디자인 단계에서 말풍선 패딩과 함께 해결
- 유저 노출 카피 전반 copywriter 경유 필요 ("광고 보고 글쓰기 3회 충전 (목)" 등 플레이스홀더 잔존)

**출시 전 처리**:
- 공개 저장소 커밋 author 이메일을 GitHub noreply로 이력 재작성 (현재 개인 이메일 노출 — 사용자 결정: 출시 전 진행)
- 시드 파일 교체: demo-tags.json(시연용) → seed-tags.json(출시 규격, §13 준수) 로드로 전환 (store.js `initStore` fetch 한 줄). **이 전환은 한 세션에서 시험 적용·검증 완료 후 되돌림** — 베타 데모 유지 목적. 재적용 시 아래 UX 항목 동반 처리 필요
- 위치 시뮬레이션 바·window.__vibemapMap·__vibemapReset·__labelDrops 제거 (index.html #dev-bar, app.js initDevBar, map.js 훅) — **실기기 위치 SDK(startUpdateLocation) 연동과 세트**. SDK 없이 개발바만 제거하면 원정 글쓰기·현장 리액션 경로가 항상 위치 null이 됨
- **출시 시드 전환 시 UX 동반 처리 (design-reviewer 지적)**: ① 초기엔 전 구 "불씨"라 롤링 칩이 `👑 —`/`🔥 —` 영구 반복 → 데이터 없을 때 전용 카피("집계 중" 등, ranking.js `rollingChips` '—' 대체) ② 랭킹 시트 빈 상태 2개 연속 시 섹션 간 여백 과다 ③ 고정석 왕관: sparse 시드에선 구당 태그 1~2개라 왕관이 흔해짐(희소성 저하) — 프로덕션 데이터 누적 시 자연 해소, 필요 시 원거리 줌 왕관 억제 검토

**기능·UX**:
- 신고 사유 UI가 브라우저 prompt() — TDS 시트로 교체
- 태그 수백 개 규모 대비: 리액션 인덱스(Map), 마커 diff 렌더, 클러스터링 검토
- 구 상세 빈 화면 독려 문구 (copywriter 경유), 불씨 목록 가독성
- 출시 전 제거: 위치 시뮬레이션 바, 데이터 초기화 버튼, window.__vibemapMap
- 자모 분리 우회(ㅅl발) 대응은 Phase 2 자모 정규화 로직 검토 (사전 _design_notes)
